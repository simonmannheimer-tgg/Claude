## Summary

**Conversation Overview**

Simon is an SEO professional working on a large-scale PLP (Product Listing Page) copywriting project for The Good Guys (TGG), an Australian electronics and appliances retailer. The project involves producing and managing category and brand category page intro copy across 1,335+ brand category URLs and 613+ non-brand category URLs, with the goal of uploading compliant copy to Contentful CMS. Simon described the project as "big, messy, all over the place" and engaged Claude across multiple sessions to consolidate, audit, and streamline the deliverables.

The core work involved three interconnected files: an updated SEO WIP tracker (replacing messy PLP intro tabs with clean two-tab versions using a four-stage status system: Not Started, Copy Drafted, Briefed for Upload, Implemented), a Contentful bulk upload file (matching the exact format of the live migration file with tabs Page Description, Page Desc. Brand Cat B1–B4), and a standalone QA checklist per page. Claude ran multiple rounds of data extraction from uploaded source files including the WIP Excel, a Contentful migration Excel, a ZIP package of Claude-generated batch CSVs and markdown files, and a rules reset ZIP containing skill reference documents. Key findings included: 479 of 843 Claude-generated brand category rows placing "The Good Guys" in S1 rather than S2, significant character count violations in pre-new-ruleset copy (non-brand mean 263 chars vs 220–250 target), 462 Brand Category Batch 1 URLs with no new compliant copy yet produced, and widespread special character issues across all files including non-breaking hyphens, en dashes, comma-periods, and curly quotes, all of which were programmatically audited and fixed.

Simon pushed back on several Claude assumptions during the session. He clarified that hard-banned word lists should not include style/quality bans, only legal or SEM-compliance items, and challenged Claude to confirm where the SEM compliance reasoning originated — correctly identifying it as Claude-inferred from a prior session rather than explicit TGG policy. He confirmed the four-stage status system (Option A), rejected the more complex QA gate structure, and directed that the bulk upload file should match the exact format of the live Contentful migration file rather than contain additional metadata columns. At session end, Simon uploaded a Claude-generated rules map document and asked for a full numbered breakdown of every rule, assumption, and guidance affecting copywriting so he could confirm each line as Y/N/conditional. The 100-item list was delivered covering character count, sentence structure, S1/S2 rules, entity rules, page type classification, hard bans, conditional bans, formatting, Australian English, brand capitalisation, batch variation rules, and pre-write workflow steps. Simon's final question challenged the sourcing of the legal/SEM ban reasoning, and Claude confirmed it was inherited Claude interpretation, not confirmed TGG policy.
