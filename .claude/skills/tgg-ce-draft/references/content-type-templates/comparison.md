# Comparison — TGG Augmentation Notes

Augments the base comparison template from `tgg-template-generator`.

---

## TGG-specific requirements

### Comparison table
Place in the first third of the article (after intro, before detailed analysis). Format:

| Feature | [Option A] | [Option B] |
|---|---|---|
| [Attribute] | [Value] | [Value] |

Minimum 4 attribute rows. Use specific values (star ratings, watts, litres) — not vague descriptors ("good", "better").

### Australian retail considerations
- Include running cost comparison where relevant (energy use per cycle/year)
- Include price range tier if it drives the decision (budget, mid-range, premium)
- Finance eligibility: note if one option is more commonly financed
- Installation: note if one option requires professional installation (adds cost)

### Recommendation callout
Mandatory. One clear paragraph that names the better option for each buyer type. Do not hedge. If one option is better for most people, say so.

Format:
> **Best for [buyer type A]:** [Option A] — because [specific reason based on an attribute].
> **Best for [buyer type B]:** [Option B] — because [specific reason].

### Structure note
Do not repeat the comparison table data in prose. The table carries the data; the body sections provide context and reasoning. Avoid "As you can see in the table above…"
